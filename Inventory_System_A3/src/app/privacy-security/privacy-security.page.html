Privacy & Security Security Commitment This application places the highest priority on the security and privacy of your inventory data. Below is a detailed breakdown of our data handling practices and mobile-specific security measures. I. Data Collection & Usage

To provide seamless inventory management, we process data strictly for operational purposes:

  • Inventory Data: Item Name, Category, Quantity, Price, Supplier, etc. This is essential for core CRUD operations.
  • Network Diagnostics: Server API Interaction logs for troubleshooting.

Privacy Guarantee: We adhere strictly to data minimization. We do not request device permissions like Camera, Microphone, or Geolocation.

II. Transmission & Backend
  • HTTPS Encryption: All communications with the server use HTTPS, preventing Man-in-the-Middle (MitM) attacks on public Wi-Fi.
  • Server-Side Validation: Data undergoes strict validation on the remote database to ensure integrity.
  • Protected Operations: Destructive API operations have fail-safes (e.g., the "Laptop" item is locked from deletion).
 III. Mobile-Specific Security

We utilize modern mobile architecture features to protect your data:

  • App Sandboxing: This application operates within the iOS/Android secure sandbox, isolating data from other unverified apps.
  • Local Storage Limitation: Sensitive inventory pricing is not stored unencrypted in `localStorage` to prevent access if the device is compromised.
  • Device-Level Security: Users are strongly advised to enable OS-level security (PIN, FaceID, TouchID) to protect unauthorized app access.
